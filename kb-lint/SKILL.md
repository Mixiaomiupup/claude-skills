# 知识库体检

对 Obsidian 单 vault 三层结构做五种系统性检查，输出体检报告。

## Vault 路径

| 层 | 路径 | 检查重点 |
|----|------|---------|
| raw/ | `~/Documents/obsidian/mixiaomi/raw/` | 素材质量、重复内容 |
| notes/ | `~/Documents/obsidian/mixiaomi/notes/` | 知识一致性、概念完整性 |
| wiki/ | `~/Documents/obsidian/mixiaomi/wiki/` | 交叉引用、来源时效性、孤立页 |

## 触发

用户说"体检"、"知识库检查"、"lint"、"扫一下知识库"。

## 五种检查

### 1. 矛盾检测

跨文章找数据/观点冲突。

**做法**：
1. 按 category 分组，读取该 category 所有文章的 frontmatter（title、summary、tags）
2. 把同主题文章的 summary 分批喂给 LLM
3. 让 LLM 找：同一事实的不同说法、互相矛盾的结论、数据不一致

**输出**：`⚠️ 矛盾：「A 文章」说 X，但「B 文章」说 Y`

### 2. 缺口发现

找被多次提及但没有专门文章的概念。

**做法**：
1. 扫描所有文章正文，提取 `[[wikilink]]` 引用和高频关键词
2. 找被引用但不存在对应文章的概念
3. 找出现在 3+ 篇文章 summary 中但没有独立文章的主题

**输出**：`📝 缺口：「多模态」被 5 篇文章提到但没有专门文章`

### 3. 关联发现

跨 category 发现未标注的联系。

**做法**：
1. 从不同 category 各取文章的 summary 和 tags
2. 让 LLM 找跨领域的共同主题、互补观点、因果链条

**输出**：`🔗 关联：「A」（编程范式）和「B」（AI思考）都在讲 agent 编排模式`

### 4. 过时清理

标记时效性内容。

**做法**：
1. 筛选 `date` 超过 3 个月的文章
2. 读取正文，让 LLM 判断是否含时效性断言（"最新"、"目前最强"、"刚发布"等）
3. 对照当前知识判断是否已过时

**输出**：`🕐 过时：「C 文章」说"GPT-4 是最强模型"（2024-03），已不准确`

### 5. 孤立页检测

找 wiki/ 层中没有任何入链的页面（orphan pages）。

**做法**：
1. 扫描 wiki/ 下所有 `.md` 文件，建立文件名集合
2. 扫描整个 vault（raw/ + notes/ + wiki/）所有文章正文，提取 `[[wikilink]]` 引用
3. 找出 wiki/ 中从未被任何文章引用的页面
4. 排除 MOC（Map of Content）和索引页等特殊页面

**输出**：`🏝️ 孤立页：wiki/「概念 X」无入链，考虑从相关文章添加引用或合并`

**背景**：Karpathy 建议知识网络中每个节点都应有入链，孤立页意味着知识未被有效串联。

## 执行策略

### 分批处理

知识库可能有几百篇文章，不能一次全扫：
- 用户说"体检" → 问要检查哪个层（raw/notes/wiki）或 category，或"全部"
- "全部"时按层逐个扫：raw/ → notes/ → wiki/，每层内按 category 分轮
- 每轮只读 frontmatter + summary，必要时才读正文（避免 context 爆炸）
- wiki/ 层额外执行：交叉引用完整性检查、来源时效性检查、孤立页检测

### 流程

```
选定范围 → 扫描三层（raw/ → notes/ → wiki/）
→ 读取文章 frontmatter + summary
→ 分批做五种检查（wiki/ 层额外做孤立页检测）
→ 汇总报告 → 存到 meta/lint/ → 展示摘要
```

### 报告存储

- 路径：`~/Documents/obsidian/mixiaomi/meta/lint/YYYY-MM-DD-<scope>.md`
- scope 示例：`全部`、`模型前沿`、`AI思考`
- 格式：按严重程度排序，每条附原文引用和建议操作

### 报告模板

```markdown
# 知识库体检报告

**范围**: {category 或"全部"}
**日期**: YYYY-MM-DD
**文章数**: N 篇

## ⚠️ 矛盾（N 条）

1. **「文章 A」vs「文章 B」**
   - A 说：...
   - B 说：...
   - 建议：核实哪个正确，更新另一篇

## 📝 缺口（N 条）

1. **「概念名」**（被 N 篇文章提到）
   - 相关文章：A、B、C
   - 建议：考虑写一篇专题

## 🔗 关联（N 条）

1. **「文章 A」↔「文章 B」**
   - 共同主题：...
   - 建议：在两篇文章中互相添加 wikilink

## 🕐 过时（N 条）

1. **「文章 C」**（YYYY-MM-DD）
   - 过时内容："..."
   - 当前事实：...
   - 建议：更新或标注

## 🏝️ 孤立页（N 条）

1. **wiki/「概念 X」**
   - 入链数：0
   - 潜在关联文章：A、B（内容相关但未引用）
   - 建议：从相关文章添加 wikilink，或合并到其他页面
```

## 用户确认

报告展示后，询问用户要不要执行建议操作：
- 添加 wikilink → 直接编辑文章
- 更新过时内容 → 提示具体修改
- 补写缺口文章 → 走 `article-gen` skill

不要自动修改文章，必须确认后操作。
