# 横纵分析法 Deep Research

> 方法论来源：数字生命卡兹克。融合语言学历时-共时分析（Saussure）、纵向-横截面研究设计、案例研究法与竞争战略分析。

## 与 analyze 的区分

| 维度 | analyze | deep-research |
|------|---------|---------------|
| 定位 | 快速内容分析/话题调研 | 万字级深度研究报告 |
| 篇幅 | 几百~几千字 | 10000-30000字 |
| 结构 | 灵活 | 严格的纵向+横向+交汇三段式 |
| 产出 | 对话内回答 | 保存为 Obsidian 文章 |

用户只说"分析一下" → analyze。用户说"横纵分析"、"深度研究"、"写个研究报告"、明确要求全面/系统/完整 → 本 skill。

## 适用研究对象

- **产品/工具**: Cursor、Claude Code、Manus Agent
- **公司/组织**: Anthropic、字节跳动、OpenAI
- **技术概念**: MCP协议、RAG、Agent框架
- **人物**: 行业关键人物的职业轨迹与同期人物对比

## 执行流程

### Phase 1: 确认研究对象与范围

1. 明确研究对象（用户可能直接给出，也可能需要澄清）
2. 快速判断研究对象的类型（产品/公司/技术/人物），这会影响纵向和横向分析的具体维度
3. 告知用户预计产出的报告结构和篇幅

### Phase 2: 信息搜集

这是报告质量的地基。用所有可用工具多轮搜索，宁多勿少。

**搜索策略**（按研究对象类型调整关键词）：

```
第1轮：研究对象本身
  → "{名称}" / "{名称} 是什么" / "{名称} 创始人/团队"
  → "{名称} founding story" / "what is {名称}"

第2轮：发展历程
  → "{名称} 发展历程/融资/版本更新"
  → "{名称} history timeline funding"
  → "{名称} 2023" / "{名称} 2024" / "{名称} 2025"（按年份搜）

第3轮：竞品与行业格局
  → "{名称} vs {竞品}" / "{名称} 竞品/alternatives"
  → "{赛道名称} landscape / market map / comparison"

第4轮：用户口碑与深度评价
  → "{名称} 评测/体验/优缺点"
  → "{名称} review reddit / hacker news"

第5轮：补充搜索
  → 根据前4轮发现的线索深挖（关键人物、关键事件、争议）
```

**工具选择**（遵循 CLAUDE.md 搜索工具优先级）：
- 通用搜索：anyweb / tavily_search / WebSearch
- 查看具体网页：anyweb 优先，降级 tavily_extract
- 中文平台内容：anyweb
- 综合调研：tavily_research
- GitHub 相关：gh CLI

每轮搜索后简要记录发现的关键信息，避免重复搜索。

### Phase 3: 撰写报告

信息搜集完成后，按以下结构撰写完整报告。

---

#### 一、纵向分析（Diachronic / Longitudinal）

沿时间轴，完整还原研究对象从诞生到现在的发展全貌。

**必须覆盖的维度**：

1. **起源追溯** — 诞生背景、底层技术/理念/需求、创始团队或核心推动者、当时的行业环境
2. **诞生节点** — 首次发布/成立/提出的时间，最初的形态和定位
3. **演进历程** — 按时间顺序梳理所有关键节点：重大版本更新、融资事件、团队变动、战略转型、技术架构变化、用户规模里程碑、重大合作或收购、公关危机或争议事件
4. **决策逻辑** — 在每个关键节点上，尽可能还原决策背后的原因。为什么选了A而不是B？当时面对的约束条件是什么？

**篇幅**: 6000-15000字。历史越长节点越多的靠近上限，新生事物靠近下限。核心原则是把故事讲完整、讲透，宁可写长写细，也不要蜻蜓点水。

---

#### 二、横向分析（Synchronic / Cross-sectional）

以当前时间点为切面，将研究对象与同赛道的竞品/同类进行全面对比。

**首先判断竞品情况**：

- **场景A：无直接竞品** — 分析为什么没有竞品（品类太新、壁垒太高、还是市场太小），未来最可能从哪个方向冒出竞争者，有没有间接替代方案
- **场景B：少量竞品（1-2个）** — 逐一深入对比，每个竞品展开详细分析
- **场景C：竞品充分（3个及以上）** — 选取最具代表性的3-5个对比，其余简要提及

**对比维度**（根据研究对象类型灵活调整）：

1. **核心差异对比** — 技术路线/方法论、产品形态/商业模式、目标用户/适用场景、核心优势与短板、定价/资源/规模
2. **用户视角** — 真实用户口碑、社区评价中最多被提及的优缺点、实际使用方式和官方定位有没有偏差
3. **生态位分析** — 在赛道版图中占据什么位置，填补了什么空白还是在正面竞争
4. **趋势判断** — 竞争格局中的走向、机会和风险

**篇幅**: 3000-10000字。场景A约3000字，场景C每个主要竞品至少1500字。

---

#### 三、横纵交汇总结

把纵向发展脉络和横向竞争格局结合起来，给出对研究对象当前所处位置和未来走向的综合判断。

这是报告的精华段——不是前面内容的缩写版，而是新的、综合性的洞察。

**篇幅**: 1500-3000字。

---

### 写作风格（贯穿全文）

这是一篇让人能从头读到尾的深度研究，不是冷冰冰的咨询报告：

1. **可读性优先** — 像优质深度报道或非虚构特稿，有节奏感和画面感
2. **叙事驱动** — 纵向部分要有故事弧线和起承转合，不要写成流水账（"2023年1月发布了A，2023年3月发布了B"）
3. **观点建立在事实之上** — 鼓励给出判断和洞察，但每个观点必须有事实支撑。先摆事实，再给判断。推测明确标注
4. **用人话写** — 避免"赋能""抓手""打造闭环"等套话，用具体的细节和例子
5. **对比有温度** — 讲清楚每个竞品"活成了什么样"，用户选它的真实理由，不只是罗列功能差异

### 格式要求

1. 先纵向分析，再横向分析，最后横纵交汇
2. 纵向以时间叙事为主线，保持可读性
3. 横向可以用对比表格辅助，但核心分析必须是文字论述
4. 所有信息尽可能标注来源或时间节点
5. 无法确认的信息明确标注为推测或未经证实

### Phase 4: 保存到 Obsidian

报告完成后，使用 article-gen skill 保存到 Obsidian vault：
- type: `research`
- 目录: `raw/具身动态/`（具身/机器人领域）或 `raw/AI/`（AI领域）或根据主题选择合适目录
- frontmatter 包含 `research_subject`、`analysis_method: 横纵分析法`

如果用户没有明确要求保存，先在对话中输出报告，询问是否保存。
