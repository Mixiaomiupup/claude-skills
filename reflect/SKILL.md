# Reflect: 复盘与周回顾

## Iron Law

```
不记录不开下一笔。不追溯不算复盘。
```

复盘不是写日记，是**更新知识库**。每次复盘必须追溯到一条验证记录——要么更新它，要么发现缺失并创建它。

## When to Use

**事件复盘**：交易亏损、项目出问题、标定失败、面试翻车、会上说了不该说的话——任何"坏结果"
**周回顾**：每周五，10 分钟，看这周的验证记录和复盘记录

## Mode: 事件复盘

### Phase 1: 无判断记录

**Iron Law: 只记录事实，不写评价。**

从用户的描述中提取信息，预填模板，让用户补充：

```markdown
## 复盘：[事件一句话描述]

**日期**：YYYY-MM-DD
**事件**：[从用户描述中提取]

### 四格表

| 什么时候 | 我做了什么决定 | 当时依据是什么 | 实际发生了什么 |
|---------|-------------|-------------|-------------|
| [预填] | [问用户] | [问用户] | [预填] |

### 归因

| 外因（我控制不了的） | 内因（我能控制的） |
|---|---|
| [预填一条] | [问用户：至少一条] |

### 行动项

下次我会：___（一条具体行动，不是"更小心"）
```

**Claude 预填**已知信息（用户说了什么事、什么结果），**问用户** 3-4 个问题补齐剩余字段。关键是**内因那一条**——用户必须自己说出来。

### Phase 2: Diff 分析

用软件工程的 diff 思维帮用户诊断：

1. **预期 vs 实际**：预期结果是什么？实际结果是什么？
2. **找参照**：有没有类似的成功案例？和成功案例逐项对比。
3. **假设**：最可能的原因是什么？用一句话陈述假设。

不是所有复盘都需要 diff 分析——交易复盘主要用四格表和归因，技术复盘（标定、代码）才需要 diff。

### Phase 3: 追溯关联

**Iron Law: 每次复盘必须追溯到知识库。**

问自己：这次出错，和知识库里的哪条记录有关？

| 情况 | 行动 |
|------|------|
| 有相关验证记录但内容不够 | 更新那条记录，加排查线索 |
| 没有相关验证记录 | 说明这个领域还没建立理解，建议走 learn skill |
| 有验证记录但没查就行动了 | 记录：下次行动前先 ob 搜一下 |
| 验证记录是对的但现实不同 | 更新记录：加"注意：实际中还要考虑___" |

### Phase 4: 存储

保存到 `~/Documents/obsidian/mixiaomi/notes/洞察/复盘/YYYY-MM-DD-[事件关键词].md`

Frontmatter:

```yaml
---
title: "复盘：[事件]"
type: notes
date: YYYY-MM-DD
category: AI思考/工作流
tags: [复盘, 相关领域]
status: enriched
linked_records: [] # 关联的验证记录路径
---
```

## Mode: 周回顾

每周五触发。Claude 自动扫描本周产出，整理清单，用户花 10 分钟看一眼。

### 扫描内容

1. **本周新增的验证记录**（`notes/学习笔记/验证记录/`，按 date 过滤本周）
2. **本周的复盘记录**（`notes/洞察/复盘/`，按 date 过滤本周）
3. **知识库活跃度**（本周有没有新文章、新笔记）

### 输出格式

```markdown
## 周回顾 YYYY-MM-DD

### 学了什么
- [列出本周验证记录，每条一行：概念名 + 验证层次]
- 如果为空：⚠️ 本周没有新的验证记录

### 复盘了什么
- [列出本周复盘记录，每条一行：事件 + 内因]
- 如果为空：本周没有需要复盘的事件（或者有但没记）

### 模式识别
- 复盘记录积累到 5 条以上时：扫描内因列，找重复模式
- 例如：连续 3 次内因都是"没提前查验证记录"→ 说明"行动前先查"的习惯还没养成

### 下周一件事
- 基于本周回顾，建议下周最值得做的一件事
```

用户确认后保存到 `notes/洞察/周回顾/YYYY-MM-DD.md`。

## Red Flags

| 你脑子里的话 | 实际含义 | 该做什么 |
|-------------|---------|---------|
| "下次再复盘吧" | 逃避机制启动了 | 现在填，10 分钟以内 |
| "这次情况不一样" | 上次也这么说的 | 打开上次复盘记录对比 |
| "都是外因" | 至少一条内因 | 写不出来就问 Claude 帮你找 |
| "我知道问题在哪" | 知道但没记录 = 下次还犯 | 写下来 |
| "太小的事不值得复盘" | 小事的模式和大事一样 | 填四格表，2 分钟 |

## Gotchas

| What happened | Rule |
|---------------|------|
| 复盘变成了自我批判 | 四格表不写评价。"我做了什么决定"不是"我犯了什么错" |
| 内因全是"不够努力" | 不具体的内因不算。"没设止损"比"不够小心"有用一百倍 |
| 复盘记录写了但没追溯 | 没有追溯的复盘是断的。必须关联到知识库的一条记录 |
| 周回顾变成长篇总结 | 10 分钟上限。清单格式，不写散文 |
