# Research 类型——公司调研

给一个公司名，从零开始调研，输出一份让读者全面了解这家公司产品和能力的报告。

## 触发词

"调研 XX"、"帮我了解 XX"、"XX 是什么公司"

## 采集流程

research 是 article-gen 唯一自带数据采集阶段的类型。采集完成后才进入 enrichment 写作。

### Step 1: 三路发现（并行）

从公司名出发，三个渠道同时搜索，尽可能全面覆盖：

1. **WebSearch**：搜"公司名 + AI/robotics/产品"，拿到官网、融资新闻、媒体深度报道 URL
2. **tavily_search**：搜"公司名"、"公司名 + CEO/founder"、"公司名 vs 竞品"，补充 WebSearch 遗漏
3. **anyweb + Grok**（可选，当 1+2 信息不足时）：向 Grok 提问"Tell me about [公司名], their products, team, funding, and how they compare to competitors. Include tweet URLs."

三路结果去重合并，提取以下关键 URL：
- 官网（About/Team/Product 页面）
- 创始人/核心员工 X 账号
- 融资新闻
- 媒体深度报道
- 竞品信息

### Step 2: 读全文（anyweb read）

对 Step 1 发现的 URL 逐个读取全文：

```bash
# 官网页面
anyweb --json read "https://公司官网/about"
anyweb --json read "https://公司官网/product"

# 媒体报道
anyweb --json read "https://techcrunch.com/xxx"

# X 账号推文（需要登录态）
anyweb --chrome --json read "https://x.com/公司官方账号"
anyweb --chrome --json read "https://x.com/创始人账号"
anyweb --chrome --json read "https://x.com/核心员工账号"
```

**X 推文重点关注**：
- 创始人自己怎么描述公司和产品
- 发布的 demo 视频和性能数据
- 同行/KOL 怎么评价（搜"公司名" OR "@创始人"）

### Step 3: 数据屏障

**所有数据源返回后才开始写作**。不要边采边写。

检查清单：
- [ ] 官网关键页面已读取
- [ ] 创始人/公司 X 账号推文已读取
- [ ] 至少 2 篇媒体报道已读取
- [ ] 竞品信息已搜集

## 采集清单

写作前必须采集到以下信息（有就写，没有标注"未公开"）：

- **产品是什么**：解决什么问题、面向什么场景、做到什么程度（demo 数据、性能指标）
- **技术路线的核心选择**：为什么走这条路而不是别的
- **创始人/核心团队自己怎么描述**：X 推文、博客原话（引用原文）
- **同行/KOL 怎么评价**：正面和负面都要
- **跟同赛道的其他公司比，差异在哪**
- **商业进展**：融资轮次/估值、客户/合作伙伴、落地场景
- **团队背景**：创始人学术/工业经历、核心成员来源

## 写作原则

**没有固定模板**。按采集到的内容自然组织，但遵循以下原则：

- **产品和能力放最前面**。读者最想知道的是"这家公司到底能干什么"，而不是融资了多少钱
- **不要固定标题**（如"公司概况"、"核心团队"），按内容自然分段，用加粗关键词起段
- **多引原话**。创始人/同行的推文原文比你的总结更有信息量。格式：`> "原话内容" —— @账号`
- **竞品对比用叙述**，不用表格（除非维度超过 4 个）。重点讲差异，不要罗列相同点
- **团队履历穿插在上下文中**，不要单独列表。比如讲技术路线时顺带提"创始人 XX 在 DeepMind 做了 5 年机器人，这解释了为什么..."
- 遵循 article-gen 通用写作风格约束（不堆指标、术语不硬翻、不格式化小标题）
- 遵循翻译术语标准（中文为主体，只有业内通用缩写才保留英文）

## 存储

- **路径**：`raw/{category一级}/公司名 - 公司调研.md`
- **不自动发飞书**，只存 Obsidian
- **frontmatter type**：`research`
- **source**：官网 URL

## 与 analyze skill 的区别

| | analyze | research |
|---|---|---|
| **输入** | 给 URL | 给公司名 |
| **采集** | 不采集，直接读给定 URL | 自己发现 URL 并采集 |
| **产出** | 分析结果（不保存） | 完整报告（存 Obsidian） |
| **深度** | 单篇内容分析 | 多源综合调研 |

用户说"帮我看看这篇" → analyze。用户说"帮我了解这家公司" → research。
