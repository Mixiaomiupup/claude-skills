# news 类型风格参考

范文：`raw/模型前沿/Anthropic - Project Glasswing与Claude Mythos.md`

## 标题

**结构：事实 + 观点，一句话。**

- 前半句说发生了什么（事实），后半句说为什么重要（观点/态度）
- 用冒号或逗号分隔，不超过 30 字
- 不堆砌信息，不罗列参与方/数字

| 好 | 坏 |
|---|---|
| Claude Mythos终于来了，但Anthropic决定不卖：先让防御者赢一次 | Anthropic发布「太强不敢公开」的AI模型Mythos，联合12家巨头启动Project Glasswing网络安全计划 |
| GPT-2 以来首次：一个 AI 模型强到不敢公开发售 | AI找漏洞已超越人类专家：Anthropic的Mythos太强不敢公开，20年网络安全攻防平衡正在被改写 |

## 正文结构

```
开头（1-3 段）
  - 一句话说清发生了什么
  - 放历史坐标（这件事在行业里的位置）
  - 简介核心产品/概念

前情/背景（如有）
  - 客观交代来龙去脉
  - 引用当事方原话

面向非专业读者的科普（如需要）
  - 用类比解释专业概念（"相当于 Anthropic 版的 ChatGPT"）
  - 用表格做结构化对比

核心内容（数据/能力/事件）
  - 数字用表格呈现，不散落在段落里
  - 关键数字加粗 + 解读（不只列数字，要说"这意味着什么"）
  - 插入官网/官方截图佐证

多方观点
  - 按观点分组（防御者视角/攻防视角/质疑视角），不按人分组
  - 引用原话用 blockquote，附身份
  - 插入关键推文截图

结尾
  - 总结"为什么重要"，2-4 个加粗要点
  - 引用一句官方原话收尾
  - 不用"综上所述"等套话
```

## 语气和措辞

**客观科技报道，不是故事会。**

| 规则 | 好 | 坏 |
|------|---|---|
| 不用叙事腔 | "3 月 26 日，Fortune 记者发现…" | "故事要从 12 天前说起" |
| 不堆社交指标 | "Kevin Roose 在推文中指出" | "Kevin Roose 的推文获得 4857 赞" |
| 不煽情 | "说明底层能力出现了不连续的跳跃" | "这简直是令人震惊的飞跃！" |
| 术语不硬翻 | "System Card（模型的出厂安全评估报告）" | "系统卡片" |
| 给读者补上下文 | "OpenBSD 被公认为安全性最强的操作系统之一" | 直接说"OpenBSD 27 年老漏洞"假设读者知道 OpenBSD |
| 引用原话佐证 | 用 blockquote 引 Anthropic/合作方原话 | 用自己的话复述官方说法 |

## 术语处理

- 业界通用英文术语保留原文，首次出现时括号加中文解释：`System Card（模型的出厂安全评估报告）`
- 之后直接用英文：`System Card 披露…`
- 不硬翻的典型术语：System Card, zero-day, sandbox escape, privilege escalation, SWE-bench, CyberGym
- 可以翻的：vulnerability → 漏洞, exploit → 漏洞利用, alignment → 对齐

## 配图

- **封面优先用官方图**（OG image、官网 hero），不自己生成
- **官网截图**：只截内容区域（深色/白色背景的图表），去掉导航栏和侧边栏
- **推文截图**：用 x-feed 标准流程（HIDE_JS + getBoundingClientRect + Pillow 动态裁剪），不硬编码坐标
- 截图插在对应内容之后，不堆在文章开头或末尾
